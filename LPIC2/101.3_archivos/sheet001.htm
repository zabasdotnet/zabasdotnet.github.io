101.3 W3 Change runlevels / boot targest and shutdown or reboot system
/etc/inittab Se define el comportamiento del proceso init y de los runlevels en el fichero /etc/inittab. 
shutdown El comando más correcto, más propio y más seguro para parar el sistema es shutdown.
init El proceso init es el padre de todos los procesos. Siempre tiene el PID 1.  Su configuración está en el fichero /etc/inittab.
/etc/init.d/ Se controlan los servicios (inicio, parada, reinicio, estatus, etc.) mediante scripts presentes en el directorio /etc/init.d.
telinit Cambiar de nivel después de arrancar la máquina.
systemd El init systemd permite acelerar el arranque de servicios, siempre basado en eventos y de forma asíncrona.
systemctl El comando systemctl permite controlar la ejecución de servicios.
/etc/systemd/ Para modificar el nivel de ejecución predeterminado, debe crear un enlace simbólico llamado default.target a la unidad objetivo deseada.
/usr/lib/systemd/ Ruta donde se  encuentran los ficheros de las unidades.
wall Escribe un mensaje a todos los usuarios.
