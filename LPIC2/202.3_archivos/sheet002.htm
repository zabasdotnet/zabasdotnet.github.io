Alternate Bootloaders Volver
Aparte de GRUB, existen otras formas de cargar un núcleo Linux.
Los principales son:
LILO: antiguo programa de Linux de carga del núcleo.
ISOLINUX: para cargar Linux desde un CD/DVD o una llave USB.
PXE: para cargar un nuevo sistema operativo desde un servidor de red.
a. LILO
LILO (LInux LOader) es el programa original de carga del núcleo Linux.
Hace tiempo era el cargador por defecto de las diferentes distribuciones, antes de ser sustituido progresivamente por GRUB, más flexible y potente.
Sin embargo, todavía es posible encontrarlo funcionando en servidores de producción antiguos.
Se desaconseja utilizarlo en nuevas instalaciones, excepto en casos concretos.
La configuración de LILO se realiza en un archivo de texto, generalmente /etc/lilo.conf.
En este archivo se especifican algunas opciones generales (disco de arranque, tiempo antes del arranque por defecto, posible contraseña), así como los diferentes núcleos que se pueden cargar, con sus opciones de carga.
LILO permite cargar otros sistemas diferentes de Linux.
Se puede proteger la carga de una o todas las imágenes con una contraseña, obligatoria o necesaria solo en caso de modificación de las opciones de carga.
Cada vez que el archivo de configuración se modifique, hay que actualizar los archivos reales utilizados por la carga mediante el comando /sbin/lilo que modificará el cargador real en el sector de arranque del disco de inicio.
En el arranque, LILO muestra, según su configuración, una línea de comandos, un menú de arranque en modo texto o un menú en modo gráfico.
Es posible modificar, a nivel de línea de comandos, las opciones de arranque específicas (en algunos casos con la condición de introducir la contraseña).
*archivo /etc/lilo.conf:
# Opciones generales: 
# Disco utilizado para el arranque 
boot = /dev/sda 
# Modo de gestión del arranque: con el símbolo de elección 
prompt 
# Imagen por defecto 
default = linux 
# Espera antes de arranque por defecto, de décimas de segundo 
timeout = 30  
  
# Selección del menú de imágenes  
# Archivo núcleo a cargar 
image = /boot/vmlinxxxxz   
# Etiqueta del menú 
label = linuxvXXXX  
# Partición raíz 
root = /dev/sda2   
# Otra imagen que se puede cargar 
image = /boot/vmlinyyyyz   
label = linuxvYYYY   
root = /dev/sda2
b. ISOLINUX
ISOLINUX se utiliza para crear CD/DVD de arranque, en el formato estándar ISO9660.
Forma parte del paquete syslinux, que es un conjunto de componentes que gestionan métodos de carga del núcleo alternativos (CD/DVD, PXE, etc.).
La idea es preparar en un directorio los diferentes archivos que se han de almacenar en la imagen ISO de arranque, y a continuación generar esta imagen mediante el comando mkisofs.
El archivo resultante se puede grabar en un CD/DVD.
Se puede montar este archivo para comprobar su contenido mediante el comando mount especificando como tipo de sistema de archivos iso9660.
También es posible crear llaves USB de arranque, copiando el archivo imagen ISO obtenido en la llave.
Lo más sencillo es partir de una imagen ISO de arranque que ya exista (un CD de instalación Linux por ejemplo), montarlo en el sistema de archivos.
A continuación, modificar su contenido en función de nuestras necesidades y generar la nueva imagen con el comando mkisofs.
c. Arranque PXE
PXE (Pre-boot eXecution Environment) es un protocolo de arranque de un sistema a través de la red.
Para poder utilizarlo, la tarjeta de red tiene que ser compatible con este protocolo (la mayoría de las tarjetas modernas lo son) y que se active esta funcionalidad en la BIOS del equipo.
Este método de arranque necesita un servicio DHCP y un servicio TFTP (Trivial File Transfer Protocol) activos en la red y configurados para la descarga remota de un núcleo Linux.
Linux puede jugar estos dos roles, permitiendo así arrancar equipos desde la red.
A menudo esta técnica se utiliza para hacer instalaciones de sistemas a través de la red.
*Principios básicos:
Hay que configurar un servidor DHCP con dos opciones para los clientes TFTP, especificando el nombre del archivo de arranque en el servidor TFTP, así como la dirección de este servidor TFTP.
A nivel del servidor TFTP, hay que copiar en su directorio de trabajo los archivos necesarios para el arranque del cliente, comenzando por el archivo indicado en las opciones del servidor DHCP.
Instalación de una distribución Debian a través de la red:
* Equipo que se ha de instalar
Configurar la BIOS para arrancar desde la red (modo PXE).
* Servidor DHCP
Crear o modificar un ámbito de direcciones para los equipos que se han de instalar en PXE, con dos opciones suplementarias:
next-server w.x.y.z ; #Dirección IP del servidor TFTP 
filename "pxelinux.0"; # Archivo de arranque en el servidor TFTP
* Servidor TFTP
Descargar desde un sitio Debian el archivo de instalación de red Debian: netboot/netboot.tar.gz
Descomprimir este archivo en el directorio de trabajo del servidor.
Deberían estar los archivos y directorios siguientes:
debian-installer
pxelinux.0
pxelinux.cfg
version.info
* Equipo en el que se ha de instalar
Arrancar la máquina.
Debería:
- Pedir y obtener una concesión del servidor DHCP.
- Pedir el archivo pxelinux.0 al servidor TFTP.
- Cargar el archivo pxelinux.0 en memoria y ejecutarlo.
- Pedir el resto de los archivos al servidor TFTP.
- Mostrar el menú de instalación.
*Proceso de inicio de GRUB
- El BIOS busca un dispositivo de inicio (como el disco duro) y de ahí pasa el control al primer sector físico del dispositivo conocido como master boot record, son los primeros 512 bytes (primer sector) del disco duro y que forman la fase 1 de GRUB.
- La fase 1 de GRUB carga la siguiente fase de GRUB, ubicada físicamente en cualquier parte del disco duro.
- La fase 1 puede cargar ya sea la fase 1.5 o, directamente, la 2.
- La fase 1.5 de GRUB, ubicada en los siguientes 30 kilobytes del disco duro, carga la fase 2. (en GRUB 2 esta fase ha dejado de existir)
- La fase 2 de GRUB (cargada por las fases 1 ó 1.5) recibe el control, y presenta al usuario el menú de inicio.
- GRUB carga el kernel seleccionado por el usuario, en la memoria y le pasa el control.
En el caso de ordenadores con sistema de arranque EFI, el firmware carga el ejecutable de GRUB directamente (fase 2), siempre y cuando esté correctamente instalado en la partición EFI y configurado en la NVRAM.
