touch
Puede necesitar crear ficheros vacíos para hacer pruebas
$ touch fichero
Utilizado con el nombre de un fichero como argumento únicamente, crea un fichero con un tamaño cero.
La creación de ficheros vacíos no es el principal uso de touch originariamente. 
Si vuelve a ejecutar el mismo comando en el fichero, observará que la fecha de modificación ha cambiado.
El manual de touch le informará de que así es posible modificar completamente la fecha y la hora de un fichero. 
Esto puede ser útil para forzar las copias de seguridad incrementales de ficheros.
