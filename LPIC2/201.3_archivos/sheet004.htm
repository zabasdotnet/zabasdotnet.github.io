Desde hace tiempo los sistemas Linux albergan en su directorio /dev el conjunto de archivos especiales para todos los dispositivos administrables por el núcleo.
Con la llegada del núcleo 2.6 apareció el servicio udev, cuya tarea es administrar dinámicamente la creación de archivos especiales cuando se detecta un nuevo periférico.
Desde el punto de vista del usuario normal, el servicio udev trabaja en segundo plano y es mejor no preocuparse: los archivos especiales están presentes cuando se necesitan y no hay razón alguna para querer cambiar este funcionamiento.
Sin embargo, el administrador o el usuario avanzado pueden crear reglas de comportamiento que desencadenarán acciones en función de los eventos relacionados con el almacenamiento.
La ubicación de estas reglas se precisa en el archivo de configuración de udev: /etc/udev/udev.conf.
En el supuesto de que no se haya informado, su valor por defecto es /etc/udev/rules.d.
*Ejemplo de regla udev:
En esta configuración estándar de una distribución Ubuntu se puede ver (después de tomarse un tiempo para interpretarla) que el sistema genera enlaces simbólicos para las distintas llamadas comunes de la unidad de discos ópticos.
usuario@estacion # cat /etc/udev/rules.d/70-persistent-cd.rules 
# This file maintains persistent names for CD/DVD reader and writer devices. 
# See udev(7) for syntax. 
# 
# Entries are automatically added by the 75-cd-aliases-generator.rules 
# file; however you are also free to add your own entries provided you 
# add the ENV{GENERATED}=1 flag to your own rules as well. 
# CD-ROM (pci-0000:00:01.1-scsi-1:0:0:0) 
SUBSYSTEM=="block", ENV{ID_CDROM}=="?*", ENV{ID_PATH}=="pci-0000:00: 
01.1-scsi-1:0:0:0", SYMLINK+="cdrom", ENV{GENERATED}="1" 
SUBSYSTEM=="block", ENV{ID_CDROM}=="?*", ENV{ID_PATH}=="pci-0000:00: 
01.1-scsi-1:0:0:0", SYMLINK+="dvd", ENV{GENERATED}="1"
