201.3 W4 Kernel runtime management and troubleshooting Volver
Candidates should be able to manage and/or query a 2.6.x or 3.x kernel and its loadable modules.
Candidates should be able to identify and correct common boot and run time issues.
Candidates should understand device detection and management using udev.
This objective includes troubleshooting udev rules.

Key Knowledge Areas:
- Use command-line utilities to get information about the currently running kernel and kernel modules
- Manually load and unload kernel modules
- Determine when modules can be unloaded
- Determine what parameters a module accepts
- Configure the system to load modules by names other than their file name.
- /proc filesystem
- Content of /, /boot/ , and /lib/modules/
- Tools and utilities to analyze information about the available hardware
- udev rules
/lib/modules/kernel-version/modules.dep Archivo de dependecia de módulos generado por depmod.
module configuration files in /etc/ Ficheros de configuración.
/proc/sys/kernel/ Este directorio contiene una variedad de archivos de configuración diferentes que afectan directamente a la operación del kernel.
/sbin/depmod Genera el fichero modules.dep y los ficheros .map (debug).
/sbin/rmmod Quitar un módulo cargado en memoria.
/sbin/modinfo Mostrar información acerca de los módulos cargados.
/bin/dmesg Guarda todos los mensajes devueltos por el núcleo desde el arranque.
/sbin/lspci Muestra todos los dispositivos PCI.
/usr/bin/lsdev Permite obtener información sobre los periféricos reconocidos, especialmente los valores DMA, IRQ e I/O.
Estos valores se leen de los archivos /proc/interrupts, /proc/ioports y /proc/dma.
/sbin/lsmod Visualización de módulos cargados en memoria.
/sbin/modprobe Visualización de los módulos disponibles en el sistema.
/sbin/insmod Carga de un módulo en memoria.
/bin/uname Imprime información del sistema.
/usr/bin/lsusb Muestra todos los dispositivos USB.
/etc/sysctl.conf, /etc/sysctl.d/ Configuración de sysctl
/sbin/sysctl Permite modificar directamente todos los archivos que se encuentran en la estructura de directorios que cuelga de /proc/sys.
udevmonitor Supervisa casi en tiempo real los eventos del sistema.
udevadm monitor Supervisa casi en tiempo real los eventos del sistema.
/etc/udev/ Conjunto de archivos especiales para todos los dispositivos administrables por el núcleo.
