El comando ps (process status) permite obtener información sobre los procesos en curso.
Si se ejecuta solo, visualiza únicamente los procesos en curso iniciados por el usuario y desde la consola actual.
$ ps
Para obtener más información, utilice el parámetro -f.
$ ps -f
El parámetro -e da información sobre todos los procesos en curso.
$ ps -ef
El parámetro -u permite precisar una lista de uno o varios usuarios separados por una coma. 
El parámetro -g efectúa lo mismo pero para los grupos, -t para los terminales y -p para unos PID determinados.
$ ps -u root
Finalmente el parámetro -l ofrece más información técnica.
$ ps -l
A continuación presentamos el detalle de algunas columnas:
UID --> User ID, nombre del usuario.
PID --> Process ID, número del proceso.
PPID --> Parent Process ID, número del proceso padre.
C --> Factor de prioridad, cuanto más grande es el valor, más elevada es la prioridad.
STIME --> Hora de inicio del proceso.
TTY --> Nombre del terminal desde el cual se ejecutó el proceso.
TIME --> Duración de tratamiento del proceso.
CMD --> Comando ejecutado.
F --> Banderas del proceso (sale del ámbito de este libro).
S --> Estado del proceso S (sleeping) R (running) Z (zombie).
PRI --> Prioridad del proceso.
NI --> Nice, incremento para el scheduler.
