Hosts virtuales de un servidor Nginx Volver
Igual que un servidor Apache, un servidor Nginx puede albergar varios sitios web diferentes: los hosts virtuales.
1. Configuración global
Se deberá modificar el archivo de configuración.
Cada host virtual tendrá sus elementos de configuración específicos en un bloque server.
Si un servidor Nginx gestiona varios hosts virtuales, se podrá configurar uno de ellos como servidor por defecto.
2. Configuración de hosts virtuales
Existen dos técnicas de implementación de hosts virtuales:
1) los hosts virtuales en direcciones IP/número de puerto, donde el servidor proporciona un contenido diferente según la dirección IP o el número de puerto por el que se realiza la conexión.
2) los hosts virtuales por nombres de host, donde el servidor proporciona un contenido diferente en función del nombre de host que haya en la URI con la que se hace la conexión.
a. Hosts virtuales en direcciones IP/número de puerto
En esta configuración, el servidor dispone de diferentes direcciones IP y/o varios números de puerto y responde en función de en qué interfaz de red y por qué puerto reciba las consultas HTTP.
Se debe utilizar un bloque server para cada dirección IP/número de puerto utilizado.
El bloque contiene la declaración del directorio raíz del servidor, a partir del cual se cargarán las páginas del sitio.
Gracias a la directiva de inclusión include, se pueden ubicar las configuraciones de los servidores virtuales en archivos separados.
*Declaración de dos sitios virtuales en el archivo de configuración de Nginx
server { 
  listen 80 default; 
  server_name localhost; 
  root  /usr/share/nginx/html/public; 
  } 
   
server { 
  listen 8080; 
  server_name localhost; 
  root  /usr/share/nginx/html/admin; 
  }
En este ejemplo, el servidor Nginx gestiona dos servidores virtuales:
- El servidor por defecto escucha el puerto 80 de todas las direcciones del equipo.
Está asociado a las peticiones dirigidas al nombre de host localhost y sus páginas se encuentran en el directorio raíz /usr/share/nginx/html/public.
- Otro servidor escucha en el puerto 8080 de todas las direcciones del equipo.
Está asociado a las peticiones dirigidas al nombre de host localhost y sus páginas se encuentran en el directorio raíz /usr/share/nginx/html/admin.
*Archivo de configuración: declaración de hosts virtuales
server --> Declaración de un host virtual: todo lo que está en el bloque afecta al host virtual.
listen [direcciónX] [:puertoY] [default] [ssl] --> El host virtual es accesible a las peticiones que lleguen por la dirección IP direcciónX y el puerto puertoY.
                                                                                                 La opción default identifica el servidor por defecto.
                                                                                                 La opción SSL especifica que el servidor utiliza el protocolo SSL.
server_name nom1 nomN --> Nombre(s) asociado(s) al servidor.
root ruta_acceso --> Define el directorio raíz de un servidor o de una ubicación, según el bloque donde se encuentre la directiva.
index fic1 [...] ficN --> Orden de búsqueda de la página de inicio por defecto para la ubicación o el servidor, según el bloque donde se encuentre la directiva.
b. Hosts virtuales en nombre de host
El soporte de hosts virtuales por nombre de host se basa en la directiva server_name.
Cuando el servidor Nginx recibe una petición de un cliente, examina el campo de encabezado Host y a continuación lo compara con los nombres declarados por los diferentes servidores virtuales.
Selecciona el primer bloque server cuya directiva server_name incluya el nombre del host.
Si ninguna directiva corresponde al nombre, el servidor Nginx utiliza la directiva listen para determinar si la dirección/puerto corresponde a un servidor.
La directiva server_name acepta varios nombres de servidores.
 Se pueden utilizar caracteres comodín (*) y expresiones regulares.
En este último caso, hay que colocar en el patrón el prefijo ~.
*Ejemplos de directivas server_name
server_name localhost ;
server_name localhost ;
server_name www.masoc.es ;
server_name www.*.masoc.es;
server_name www.masoc.es www.empresa.com masoc.org ;
server_name  ~[mtl]asoc\..*$;
server_name www.soc.com "" ;
*Ejemplo de declaración de sitios virtuales por nombre de host
        server {  
          listen 80;  
          server_name www.conta.masoc.com;  
          root  /usr/share/nginx/html/public/conta;  
          }  
        server {  
          listen 80;  
          server_name www.marketing.masoc.com;  
          root  /usr/share/nginx/html/public/mkg;  
          }  
        server {  
          listen 80 default;  
          server_name beta;  
          root  /usr/share/nginx/html/public;  
          }
En este ejemplo, el servidor Nginx gestiona tres servidores virtuales:
- El primer servidor escucha en el puerto 80 de todas las direcciones del equipo.
Está asociado a las peticiones dirigidas al nombre de host www.conta.masoc.com y sus páginas están en el directorio raíz /usr/share/nginx/html/public/conta.
- El segundo servidor escucha en el puerto 80 de todas las direcciones del equipo.
Está asociado a las peticiones dirigidas al nombre de host www.marketing.masoc.com y sus páginas están en el directorio raíz /usr/share/nginx/html/public/mkg.
- El último servidor escucha en el puerto 80 de todas las direcciones del equipo.
Está asociado a las peticiones dirigidas al nombre de host beta y sus páginas están en el directorio raíz /usr/share/nginx/html/public/.
Se trata del servidor por defecto.
Se le dirigirá cualquier petición enviada sin nombre de host o a otro nombre que no sea www.conta.masoc.com o www.marketing.masoc.com.
*Archivo de configuración: declaración de hosts virtuales por nombre
server --> Declaración de un host virtual: todo lo que está en el bloque afecta al host virtual.
listen [direcciónX] [:puertoY] [default] [ssl] --> El host virtual es accesible a las peticiones que lleguen por la dirección IP direcciónX y el puerto puertoY.
                                                                                                 La opción default identifica el servidor por defecto.
                                                                                                 La opción SSL especifica que el servidor utiliza el protocolo SSL.
server_namenombre1 nombreN --> Nombre(s) asociado(s) al servidor. Se pueden especificar varios nombres, utilizar caracteres comodín o expresiones regulares (con el prefijo ~).
root ruta_acceso --> Define el directorio raíz de un servidor o de una ubicación, según el bloque donde se encuentre la directiva.
index fic1 [...] ficN --> Orden de búsqueda de la página de inicio para la ubicación o el servidor, según dónde esté la directiva.
