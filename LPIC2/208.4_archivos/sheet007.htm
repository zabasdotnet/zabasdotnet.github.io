Gestión de páginas dinámicas con un servidor Nginx Volver
Como sucede en Apache, Nginx puede gestionar las peticiones dinámicas, es decir las llamadas a páginas generadas dinámicamente mediante la ejecución de programas escritos en diferentes lenguajes (PHP, Perl, Python, Ruby, etc.).
Para ello, Nginx gestiona diferentes mecanismos de ejecución de programas, siendo el más utilizado FastCGI.
Cuando el servidor Nginx recibe una petición que corresponde a una página dinámica, puede transferir esta petición a un servidor de aplicaciones que utilice el protocolo FastCGI.
El servidor de aplicaciones ejecuta el programa y retorna una página HTML generada dinámicamente, que Nginx retornará al cliente.
Para ello, hay que configurar FastCGI en el servidor Nginx y utilizar servidores de aplicaciones compatibles con FastCGI.
1. Los módulos FastCGI
Existen diferentes módulos Nginx que permiten interactuar con servidores de aplicaciones:
ngx_http_fastcgi --> módulo estándar, el más utilizado.
ngx_http_scgi --> módulo estándar que utiliza un protocolo de aplicaciones más reciente que FastCGI.
ngx_http_uwsgi --> módulo estándar, solo es compatible con algunos servidores de aplicaciones.
Para cada uno de estos módulos, la mayor parte de la configuración básica se encuentra en los tres siguientes archivos de /etc/nginx:
- uwsgi_params
- scgi_params
- fastcgi_params
Basta incluir los que utilizaremos en el archivo de configuración de Nginx con la directiva include.
2. Configuración de FastCGI
Vamos a considerar la configuración para la gestión de páginas PHP de un servidor Nginx a través de FastCGI.
Para el resto de los módulos actuaremos del mismo modo.
La configuración se hace asociando un bloque de ubicación a las URI que se desea tratar con un servidor de aplicaciones (PHP, Python, etc.).
En el bloque de ubicación, se declaran algunas directivas de configuración del envío de la petición y se especifica el servidor de destino.
*Ejemplo:
location /cgi/ { 
  include fastcgi_params; 
  fastcgi_param SCRIPT_FILENAME     $document_root$fastcgi_script_name; 
  fastcgi_param QUERY_STRING $query_string; 
  fastcgi_pass localhost:9000; 
}
Cualquier petición que comience por /cgi/ se transmitirá al servidor de aplicaciones de la máquina local por el puerto 9000.
Los dos parámetros SCRIPT_FILENAME y QUERY_STRING son obligatorios:
- El primero indica dónde buscar los scripts que corresponden a la petición
 (el nombre del script, con el prefijo /cgi/, en la variable $fastcgi_script_name, precedido del directorio raíz del servidor, que se almacena en la variable $document_root).
- El segundo especifica el paso de parámetros de la URI (variable $query_string).
La directiva include permite añadir el contenido del archivo de configuración por defecto, proporcionado con Nginx.
