Configuración de Nginx con SSL Volver
El servidor Nginx gestiona los accesos seguros a través de SSL (Secure Socket Layer).
Para ello, utiliza el módulo opcional SSL, que se debe incluir en la compilación con la opción --with-http_ssl_module.
Este módulo depende de la biblioteca OpenSSL, que debe instalarse previamente en el sistema.
El módulo SSL generalmente está incluido en las versiones de las distribuciones.
Para asegurarnos, podemos utilizar el comando nginx -V.
El servidor utiliza claves y certificados, que se generan según el método visto en la sección del servidor Apache.
1. Configuración de un servidor virtual SSL
La configuración se hace mediante el argumento ssl de la directiva listen, en el bloque server:
listen 443 ssl;
El puerto asociado por defecto al protocolo SSL es el puerto 443.
La palabra clave ssl indica que el servidor virtual asociado al bloque acepta conexiones SSL.
Hay que especificar al servidor qué claves hay que utilizar para que funcione en modo SSL.
Debe disponer de su clave privada y del certificado que contendrá su clave pública.
ssl_certificate Ruta_Archivo_Certificado; 
ssl_certificate_key Ruta_Archivo_Clave;
*Ejemplo
server { 
  listen 443 ssl ; 
  server_name www.conta.masoc.com; 
  ssl_certificate /usr/share/nginx/ssl/cert.pem; 
  ssl_certificate_key /usr/share/nginx/ssl/cert.key; 
  root  /usr/share/nginx/html/public/conta; 
  }
El archivo /usr/share/nginx/ssl/cert.pem debe contener un certificado de servidor válido, y el archivo /usr/share/nginx/ssl/cert.key debe contener la clave.
Para las pruebas, es posible generar los dos archivos con el comando openssl:

2. Optimización de un servidor SSL
Para optimizar el rendimiento del servidor en modo SSL, se pueden añadir directivas específicas en el bloque server:
*keepalive_timeout  70;
Activa el modo de mantenimiento de conexión (keepalive) y fija el time-out en 70 segundos.
*ssl_session_cache   shared:SSL:10m;
Caché de gestión de sesiones activado, con una duración de mantenimiento de 10 minutos.
*ssl_session_timeout 10m;
Time-out de sesión de 10 minutos.
Además, existen otras opciones de configuración que están descritas en la documentación del módulo.
