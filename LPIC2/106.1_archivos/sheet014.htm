DISPLAY Volver
La palabra mágica en un sistema de ventanas X es DISPLAY.
Consiste en lo siguiente:
- un teclado, un ratón y una pantalla.
El display es administrado por un programa servidor, conocido como X Server.
El servidor sirve las posibilidades de visualización a otros programas que se conectan a él.
El servidor remoto conoce donde tiene que redirigir el tráfico de red X a través de la definición de la variable de entorno DISPLAY.
Generalmente apunta a un servidor de Display X que se encuentra en la máquina local.
El valor de la variable de entorno display es:
hostname:(D.S.)
 hostname el el nombre del ordenador donde se ejecuta el servidor X.
Si se omite significa que es localhost.
D es un número sequencial (normalmente 0). Puede variar si hay multiples pantallas conectadas a un ordenador.
S es el número de pantalla. Puede haber varias. Normalmente solo hay una que es 0, por defecto.
Ejemplos:
localhost:4
google.com:0
:0.0
hostname:D.S
Significa pantalla S en DISPLAY D del nombre de host hostname;
el servidor X para este Display está escuchando en el puerto TCP 6000+D.
host/unix:D.S
significa pantalla S en DISPLAY D del host host; el servidor X para este display esta escuchando en el socket de dominio UNIX /tmp/.X11-unix/XD
(así que solo se puede acceder desde host)
:D.S es equivalente a host/unix:D.S, donde host el el nombre local de la máquina.
:0.0
Significa que estamos hablando de la primera pantalla conectada a tu primer display en tu máquina local.
