Descubrir un servidor iSCSI Volver
El descubrimiento de un servidor iSCSI se hace mediante su portal de red, identificado por una dirección IP.
(Eventualmente, un puerto si no se utiliza el puerto por defecto).
La opción -m discovery del comando iscsiadm permite descubrir los destinos iSCSI expuestos por el servidor de almacenamiento iSCSI remoto.
*Ejemplo
[root@beta ~]# iscsiadm -m discovery -t sendtargets -p 192.168.0.39 
Inicio de iscsid:                                    [  OK  ] 
192.168.0.39:3260,1 iqn.2008-09.com.ejemplo:server.target1
En este ejemplo, se pregunta al servidor 192.168.0.39 enviándole los identificadores WWN clientes locales (Initiator WWN).
El servidor remoto responde indicando el WWN de un destino accesible, en este caso: iqn.2008-09.com.ejemplo:server.target1.
*Conexión a un destino iSCSI
Después del descubrimiento, hay que conectarse al destino deseado.
Si esta conexión es correcta, se guardará en la base de datos del servidor iscid y se podrá conectar automáticamente.
Las opciones -m node, -T destino, -p portal y --login del comando iscsiadm permiten conectar con un destino.
*Ejemplo
[root@beta ~]# iscsiadm -m node -T iqn.2008-09.com.ejemplo:  
server.target1 -p 192.168.0.39 --login  
Logging in to [iface: default, target: iqn.2008-09.com.ejemplo:  
server.target1, portal: 192.168.0.39,3260] (multiple)  
Login to [iface: default, target: iqn.2008-09.com.ejemplo:  
server.target1, portal: 192.168.0.39,3260] successful.
Si la conexión se ha aceptado, las unidades de almacenamiento gestionadas por el destino se asocian a archivos especiales locales en modo bloque, como si se tratase de dispositivos locales.

*Utilización del dispositivo iSCSI

Se puede utilizar como un disco entero o particionarlo.

*Mantener la conexión
Generalmente es necesario que se pueda acceder al sistema de archivos cada vez que se inicie el sistema:
*Definir que el servidor iscsid arranque automáticamente
[root@beta ~]# chkconfig iscsid on 
[root@beta ~]# chkconfig --list iscsid 
iscsid          0:off 1:off 2:on        3:on        4:on 
       5:on        6:off 
[root@beta ~]#
*Configurar el sistema de archivos en /etc/fstab
/dev/sde /var/bigdatas ext4 _netdev 0 0
La opción _netdev especifica el modo de acceso a la red del sistema de archivos.

*Desconexión de un recurso iSCSI
Si el sistema debe dejar de utilizar un dispositivo remoto iSCSI, se puede pedir su desconexión a iscsid, mediante la opción --logout del comando iscsiadm, pero después de asegurarse de que no esté montado.
*Ejemplo
iscsiadm -m node -T iqn.2008-09.com.ejemplo:server.target1 -p 
192.168.0.39 -logout
