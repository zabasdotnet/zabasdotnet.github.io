Gestión de discos iSCSI Volver
SCSI (Small Computer System Interface) es una familia de protocolos de comunicación destinados a los dispositivos de entrada/salida.
Es un protocolo cliente servidor, que permite al cliente (el sistema) enviar peticiones al servidor (el controlador SCSI), que se encarga de gestionar las unidades de almacenamiento (discos, lectores de cintas, etc.).
iSCSI (Internet SCSI) es una extensión de red del protocolo SCSI, que permite transferir comandos SCSI a través de una red TCP/IP.
De este modo se pueden gestionar a distancia accesos a discos de bajo nivel, sobre una topología de tipo Ethernet.
Este protocolo estándar está definido en la RFC 3720.
iSCSI se utiliza en el ámbito de redes de almacenamiento (Storage Area Network) en TCP/IP, que es una solución menos costosa que las redes SAN en fibra óptica.
Open-iSCSI es una implementación de este protocolo sobre Linux.
Este software permite a un sistema Linux utilizar este protocolo como cliente (acceder a los servidores de almacenamiento de disco iSCSI).
Linux también puede jugar el papel de servidor de almacenamiento iSCSI y, en este caso, la máquina gestionada por Linux pondrá algunos de sus discos a disposición exclusiva de los clientes iSCSI remotos.
a. Terminología
- Un cliente iSCSI se llama iniciador iSCSI (iSCSI Initiator).
Se identifica con un identificador único, en formato WWN (World Wide Name).
- Un destino iSCSI (iSCSI target) corresponde a una interfaz de un servidor de almacenamiento.
Se identifica con un identificador único, en formato WWN.
Un destino se puede asociar a diferentes direcciones IP e, inversamente, una dirección IP puede corresponder a varios destinos.
- Un destino iSCSI gestiona unidades lógicas de almacenamiento, identificados por un número de unidad lógica LUN (Logical Unit Number).
Cada LUN es vista por el sistema como un dispositivo físico de almacenamiento (disco duro, cartucho).
- Un portal de red (Network Portal) es un elemento iSCSI que se puede asociar a una conexión TCP/IP.
Un portal de red servidor está a la escucha de una dirección IP y de un puerto TCP.
El puerto reservado a iSCSI es el puerto 3260.
Los nombres iSCSI utilizan una nomenclatura de tipo WWN (World Wide Name), de manera que asegure que sea única:
iqn. --> prefijo que especifica el espacio de nombres (iSCSI qualified name).
AAAA-MM. --> fecha (Año-Mes) prevista que corresponderá al primer mes de asignación del nombre de dominio utilizado por la autoridad administradora de este WWN.
Dominio --> nombre de dominio, en orden inverso, de la autoridad administradora de este WWN.
:cadena_ident --> eventualmente, identificador del elemento designado para este nombre. La unicidad de este identificador es responsabilidad de la autoridad administradora del WWN.
*Ejemplo
iqn.2014-01.net.pas:cloud1.target1
Destino target1 del servidor iSCSI cloud1 del dominio pas.net, declarado desde enero de 2014.
