hdparm Volver
El comando hdparm permite consultar y configurar una gran cantidad de parámetros del disco duro.
Algunos de ellos además pueden ser peligrosos para el disco.
*Visualización de los parámetros funcionales con hdparm
alfa:~# hdparm /dev/hda 
/dev/hda: 
 multcount     =  0 (off) 
 IO_support    =  0 (default) 
 unmaskirq     =  0 (off) 
 using_dma     =  1 (on) 
 keepsettings  =  0 (off) 
 readonly      =  0 (off) 
 readahead     = 256 (on) 
 geometry      = 16644/16/63, sectors = 16777216, start = 0
Si el hardware lo soporta, los parámetros funcionales del disco pueden modificarse con la opción apropiada seguida de un parámetro numérico, en general 0 o 1.
Las opciones más comunes son c (acceso de 32 bits al disco activado o desactivado) y d (acceso por DMA activado o desactivado).
Una opción solicitada sin valor numérico asociado provocará la visualización del valor actual.
*Consulta del acceso de 32 bits
alfa:~# hdparm -c /dev/hda 
/dev/hda: 
 IO_support    =  0 (default)
*Consulta y supresión de la lectura anticipada
alfa:~# hdparm -a /dev/hda 
/dev/hda: 
 readahead     = 256 (on) 
alfa:~# hdparm -a 0 /dev/hda 
/dev/hda: 
 setting fs readahead to 0 
 readahead     =  0 (off)
*Visualización, activación y desactivación del acceso por DMA
alfa:~# hdparm -d /dev/hda 
/dev/hda: 
 using_dma     =  0 (off) 
alfa:~# hdparm -d 1 /dev/hda 
/dev/hda: 
 setting using_dma to 1 (on) 
 using_dma     =  1 (on) 
alfa:~# hdparm -d 0 /dev/hda 
/dev/hda: 
 setting using_dma to 0 (off) 
 using_dma     =  0 (off)
Otro comando: sdparm, menos común, permite una comunicación de bajo nivel con los periféricos SCSI.
Por ejemplo para realizar su desactivación para retirarlos en caliente.
