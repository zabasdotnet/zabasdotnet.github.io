XFS Volver
XFS es un sistema de archivos creado por la empresa SGI (Silicon Graphics Inc) para sus sistemas Unix IRIX. Creado en 1994, se beneficia de un largo recorrido.
Se utiliza sobre todo en entornos de producción, en servidores que gestionan grandes espacios de almacenamiento en disco.
SGI puso este sistema de gestión de archivos bajo licencia GPL en el año 2000, lo que permitió a Linux adoptarlo.
Está integrado en el núcleo de Linux a partir de la versión 2.4, y está disponible en la mayoría de las distribuciones.
Se concibió especialmente para gestionar de forma eficaz grandes volúmenes de disco y sus principales características son las siguientes:
- 64 bits:
El límite teórico máximo del tamaño de un archivo o de un sistema de archivos completo es de 8 exabytes (2ˆ63 - 1).
En la práctica está limitado por la configuración de las distribuciones (100 TB para Red Hat 6, 500 TB para Red Hat 7), pero permite pasar ampliamente los 16 TB que en la actualidad son el máximo para los sistemas de archivos de tipo ext.
- Con registro:
Las operaciones de escritura sobre los metadatos se escriben en el registro antes de ejecutarse, lo que permite garantizar la coherencia del sistema de archivos en caso de fallo físico.
Para optimizar el rendimiento, es posible posicionar el registro en un volumen físico distinto.
- E/S paralelas:
Gracias a su organización en grupos de asignación (internamente, un sistema de archivos se divide en varios subconjuntos independientes), permite realizar de forma simultánea diferentes operaciones de entrada/salida en un mismo sistema de archivos, lo que es particularmente adecuado para aplicaciones multihilo.
- Diferentes técnicas permiten asegurar un alto nivel de rendimiento y escalabilidad:
Utilización de árboles indexados (B-tree) para los metadatos, asignación diferida, preasignación para evitar la fragmentación, defragmentación en línea, asignación por extensiones, etc.
- Las herramientas de verificación y de corrección de los sistemas de archivos son muy rápidas.
- Atributos ampliados:
Permiten gestionar listas de control de acceso (ACL), además de los derechos de acceso clásicos del mundo Unix/Linux, así como atributos específicos de ciertas aplicaciones.
- Sistema de cuotas de utilización de espacio muy flexible, por grupo, usuario o «proyecto» (cuota asociada a una rama de la arborescencia).
- Compatible con snapshots:
Esta técnica de toma de imágenes del estado del sistema de archivos en un momento te permite realizar copias de seguridad en línea.
- Extensible en línea:
Se puede ampliar mientras se utiliza (pero no reducir).
- Herramientas de copia de seguridad/restauración específicas para el sistema de archivos.
Este sistema de gestión de archivos se adapta sobre todo a grandes servidores, necesita mucha memoria RAM e implica utilizar equipos fiables y seguros (con una batería o SAI).
XFS es el sistema de archivos por defecto de Red Hat 7.
En los sistemas de tipo Red Hat, el módulo que administra XFS no está incluido en las versiones de 32 bits. Por el contrario, sí está en las distribuciones Debian de 32 y 64 bits.
El administrador del sistema dispone de comandos específicos para gestionar los sistemas de archivos de tipo XFS al máximo de sus posibilidades.
*Creación de sistemas de archivos XFS
De la misma forma que en los sistemas de tipo ext, el comando genérico mkfs en realidad llama a un comando específico, mkfs.xfs, cuando se le pide crear un sistema de archivos con la opción -t xfs.
mkfs.xfs device
Donde device es el  dispositivo de modo bloque en el que se crea el sistema de archivos.
Con las opciones se pueden especificar diversos parámetros, en particular, el tamaño de los datos (si no se quiere utilizar todo el espacio del dispositivo), el tamaño y la ubicación del registro (por ejemplo en otro dispositivo), la etiqueta de identificación única del sistema de archivos (para que sea independiente del nombre del dispositivo).
También se puede crear un sistema de archivos XFS en un volumen lógico.
*Ejemplo:
[root@beta ~]# mkfs.xfs -f -d size=25g /dev/sdb 
meta-data=/dev/sdb           isize=256    agcount=4, agsize=1638400 blks 
         =                   sectsz=512   attr=2, projid32bit=0 
data     =                   bsize=4096   blocks=6553600, imaxpct=25 
         =                   sunit=0      swidth=0 blks 
naming   =version 2          bsize=4096   ascii-ci=0 
log      =internal log       bsize=4096   blocks=3200, version=2 
         =                   sectsz=512   sunit=0 blks, lazy-count=1 
realtime =none               extsz=4096   blocks=0, rtextents=0
En este ejemplo, se ha creado un sistema de archivos XFS en el disco /dev/sdb, con una zona de datos de unos 25 GB (-d size=25g), forzando la sobrescritura si ya había un sistema de archivos en el dispositivo (-f).
*Gestión de la etiqueta de un sistema de archivos XFS
El comando xfs_admin permite mostrar la etiqueta de un sistema de archivos XFS, y de modificarla siempre que se desmonte.
xfs_admin -l
Muestra la etiqueta del sistema de archivos.
xfs_admin -L descripción
Asigna la etiqueta descripción al sistema de archivos (máximo 12 caracteres).
*Ejemplo:
[root@beta ~]# xfs_admin -l /dev/sdb 
label = "" 
[root@beta ~]# xfs_admin -L datas /dev/sdb 
writing all SBs 
new label = "datas"

*Extensión de un sistema de archivos XFS
Sólo se puede aumentar el tamaño de un sistema de archivos XFS si está montado.
No se puede reducir el tamaño.
Es necesario que quede espacio libre en el dispositivo (o en el volumen lógico).
xfs_growfs -D NuevoTamaño PuntoMontaje
o
xfs_growfs -D NuevoTamaño dispositivo
Cada uno estos dos comandos aumenta el tamaño del sistema de archivos dispositivo (montado en PuntoMontaje) al nuevo tamaño especificado, expresado en número de bloques.
xfs_growfs -d dispositivo
Este comando fuerza el nuevo tamaño al máximo del espacio libre en el dispositivo.
Se puede realizar la operación incluso si se está utilizando el sistema de archivos.
*Ejemplo:
[root@beta ~]# xfs_growfs -d /dev/sdb  
meta-data=/dev/sdb           isize=256    agcount=4, agsize=1638400 blks 
         =                   sectsz=512   attr=2, projid32bit=0  
data     =                   bsize=4096   blocks=6553600, imaxpct=25  
         =                   sunit=0      swidth=0 blks  
naming   =version 2          bsize=4096   ascii-ci=0  
log      =internal           bsize=4096   blocks=3200, version=2  
         =                   sectsz=512   sunit=0 blks, lazy-count=1 
realtime =none               extsz=4096   blocks=0, rtextents=0

[root@beta datas]# df /var/datas  
Filesystem     1K-blocks  Used Available Use% Mounted on

El sistema de archivos se aumenta hasta el tamaño máximo libre en el disco /dev/sdb, o sea 7.831.552 bloques de datos, con un tamaño de bloque de 4.096 bytes.
*Copia de seguridad y restauración de un sistema de archivos XFS

Solo se puede hacer una copia de seguridad de un sistema de archivos si está montado.
Se puede realizar una copia de seguridad completa del sistema de archivos, o solo de los elementos modificados desde la última copia de seguridad, o solo de algunos archivos.
El comando gestiona niveles de copia de seguridad, lo que permite implementar estrategias de copias incrementales.
El nivel cero indica una copia de seguridad completa.
La copia de seguridad se puede interrumpir y volver a poner en marcha posteriormente.

*Defragmentar un sistema de archivos XFS
xfs_fsr
El sistema de archivos debe estar montado.
Por defecto el comando se aplica a todos los archivos de todos los sistemas de archivos XFS montados.
Se le puede pasar como parámetro la ruta de acceso de cada sistema de archivos o archivo que se ha de defragmentar.
*Ejemplo:
[root@beta datas]# xfs_fsr /dev/sdb 
/var/datas start inode=0
*Controlar un sistema de archivos XFS
xfs_check
Para evitar errores durante el control, el sistema de archivos se debería desmontar o montar en solo lectura.
*Ejemplo:
[root@beta datas]# xfs_check /dev/sdb  
xfs_check: /dev/sdb contains a mounted and writable filesystem  
  
fatal error -- couldn’t initialize XFS library  
[root@beta datas]# cd  
[root@beta ~]# umount /var/datas  
[root@beta ~]# xfs_check /dev/sdb  
[root@beta ~]#
El primer intento devuelve un error, ya que el sistema de archivos está montado.
