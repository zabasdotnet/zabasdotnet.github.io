collectd Volver
El demonio collectd es un programa open source de recogida de datos de la actividad del sistema.
Se trata de un demonio que mide a intervalos regulares (10 segundos por defecto) los valores de diferentes ordenadores que reflejan el rendimiento del sistema y los guarda en archivos.
Tiene como objetivo el análisis del rendimiento (falta de recursos, cuellos de botella) y la planificación de los recursos (control de la evolución del consumo de recursos).
collectd no permite explotar la información obtenida, pero puede producir archivos de recogida en diferentes formatos estándar que se pueden abrir con aplicaciones de presentación de datos.
Por defecto utiliza el formato RRD (Round Robin Database), muy utilizado por programas de presentación gráfica de datos estadísticos.
collectd tiene un muy buen rendimiento y consume pocos recursos.
Está escrito en C.
Creado de manera modular, dispone de numerosos plugins para interactuar con otras aplicaciones, ya sea para su control o para integrar sus resultados (Apache, DNS, MySQL, Oracle, Nagios…).
collectd puede funcionar en red, enviando sus datos a uno o varios servidores remotos (en modo push, es decir, a petición de los sistemas locales, sin polling desde un servidor).
La transferencia de datos se puede hacer en IP v4 o IP v6, en modo unicast o multicast, y, si es necesario, a través de un proxy.
También puede funcionar en modo local, sin que necesite ninguna configuración de red.

b. Configuración
La configuración por defecto permite arrancar inmediatamente el demonio.
Para una utilización más personalizada, hay que modificar el archivo de configuración, /etc/collectd.conf o /etc/collectd/collectd.conf. 
*Ejemplo:
Archivo de configuración por defecto, después de la instalación en una distribución Debian (sin los comentarios).
root@alfa:~# vi /etc/collectd/collectd.conf  
# sin los comentarios 
FQDNLookup true  
LoadPlugin syslog  
<Plugin syslog>  
        LogLevel info  
</Plugin>  
LoadPlugin battery  
LoadPlugin cpu  
LoadPlugin df  
LoadPlugin disk  
LoadPlugin entropy  
LoadPlugin interface  
LoadPlugin irq  
LoadPlugin load  
LoadPlugin memory  
LoadPlugin processes  
LoadPlugin rrdtool  
LoadPlugin swap  
LoadPlugin users  
<Plugin rrdtool>  
        DataDir "/var/lib/collectd/rrd"  
</Plugin>  
Include "/etc/collectd/filters.conf"  
Include "/etc/collectd/thresholds.conf"
Los diferentes servicios que se deben controlar se especifican mediante los plugins específicos (cpu, disco, memoria, interfaces de red, procesos, etc.).
El formato de salida es RRD y el directorio de salida de los datos es /var/lib/collectd/rrd.

En el directorio del servidor local (alfa), hay toda una serie de directorios para los diferentes recursos de los que se recoge información.
*Arranque:
El demonio se ha instalado como servicio, y se puede activar manualmente mediante el comando:
service collectd start
También se puede configurar en arranque automático.
*Ejemplo:
root@alfa:~# service collectd status 
[ ok ] collectd is running.
c. Explotación de los datos de collectd
Como hemos visto, collectd no puede utilizar directamente la información obtenida.
Así pues, hay que visualizarla con aplicaciones de presentación de datos o software específico.
Por defecto utiliza el formato RRD (Round Robin Database), que es un formato estándar para los archivos de datos estadísticos.
Se pueden generar gráficos o tablas mediante programas como RRDtool, kcollectd o drraw, o mediante programas específicos que utilizan bibliotecas especializadas (Python, Perl, C, Java…).
*collection3
El paquete collectd contiene, por ejemplo, un conjunto de scripts CGI de generación de gráficos escritos en Perl.
*Ejemplo:
Después de la instalación del paquete Debian:
root@alfa:~# cd /usr/share/doc/collectd/examples 
root@alfa:/usr/share/doc/collectd/examples/collection3# ls 
bin  etc  lib  README  share
Estos directorios se deben copiar en un directorio virtual gestionado por un servidor web.
Los scripts en Perl necesitan que previamente se instalen diferentes bibliotecas y módulos Perl (el archivo README).
*collectd-web
Vamos a mostrar una solución open source especialmente creada para utilizar los datos de collectd, a partir de cualquier servidor web (Apache, Nginx o incluso un simple servidor Python dedicado): collectd-web. 
*Instalación
La instalación se hace a partir de un archivo comprimido.
A continuación hay que copiar el árbol de directorios en un directorio gestionado por un servidor web.
Para ello, utilizaremos el servidor Apache2, configurando un nuevo servidor virtual.
*Instalación de los scripts y copia en un directorio del servidor Apache2.
root@alfa:~# wget https://github.com/downloads/httpdss/collectd-web/collectd-web_0.4.0.tar.gz
root@alfa:~# tar -zxvf collectd-web_0.4.0.tar.gz 
root@alfa:~# mv collectd-web /var/www/collectd 
root@alfa:~# chmod +x /var/www/collectd/cgi-bin/*.cgi 
root@alfa:~# chmod +x /var/www/collectd/cgi-bin/*.pl
Para que los scripts en Perl funcionen correctamente, hay que instalar la biblioteca Perl de gestión del formato RRD, así como los diferentes módulos de Perl (mediante el comando cpan install, proporcionado con Perl).
El script /var/www/collectd/check_deps.sh, que viene con collectd-web, permite comprobar que todos los módulos están correctamente instalados.
root@alfa:~# apt-get install librrds-perl 
[...] 
root@alfa:~# cpan install Config::General 
[...] 
root@alfa:~# cpan install Regexp::Common 
[...] 
root@alfa:~# cpan install HTML::Entities 
[...] 
root@alfa:~# cpan install RRDs 
root@alfa:~# cpan install JSON 
[...] 
root@alfa:~# /var/www/collectd/check_deps.sh
*Configuración del servidor virtual
vi /etc/apache2/sites-available/default 
<Directory /var/www/collectd> 
Options Indexes FollowSymLinks MultiViews 
AllowOverride All 
Order allow,deny 
allow from all 
</Directory>
*Utilización
Para acceder al programa, basta con utilizar un navegador de Internet y escribir la URL:
http://nombreServidor/collectd
La página de inicio muestra la lista de los servidores que supervisa collectd y, una vez seleccionamos uno de ellos, permite seleccionar el recurso cuyas estadísticas se quieren ver.
